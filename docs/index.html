OAuth

Up 
get /sso/authorize
Initate OAuth Authorization (ssoAuthorizeGet)
<p>Should redirect user to this page for authorization</p> <p>See also: <a href="https://www.oauth.com/oauth2-servers/accessing-data/authorization-request/">OAuth Specification on Authorization Request</a></p> <hr /> <p>After authorization, server will redirect user back to callback url specified in Client Settings with following query parameters available:</p> <ul> <li>For <a href="https://www.oauth.com/oauth2-servers/access-tokens/authorization-code-request/">Authorization Code</a> Flow: <ul> <li>state</li> <li>code</li> </ul> </li> <li>For Implicit Flow: <ul> <li>access_token</li> <li>token_type</li> <li>user_id</li> <li>state</li> </ul> </li> </ul>

Query parameters

response_type (required)
Query Parameter — Flow type. code for Authorization Code Flow or token for Implicit Flow
client_id (required)
Query Parameter — OAuth Client ID
scope (optional)
Query Parameter — <p>Describe which binding is required for this grant. Separated by whitespace.</p> <p>Valid values are 'bbs' 'minecraft' 'telegram'</p> <p>If omitted, there will be no limit on this grant.</p>
state (required)
Query Parameter — Random string to prevent request forgery

Responses

200

Load authorization page
